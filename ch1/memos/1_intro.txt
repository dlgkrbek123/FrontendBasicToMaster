웹이 성장해서 복잡해져감에 따라 프론트엔드와 백엔드가 분리
클라이언트 영역은 UI를 개발하며 아래의 작업을 포함한다.

 1. 사용자가 입력한 데이터를 비즈니스 로직에 따라 처리
 2. 서버와 통신해서 얻어온 데이터를 출력

---------------------------------------------------------------------

웹 1.0 
- 단방향 정보 제공 / 정적인 서비스

웹 2.0
- 사용자가 직접 정보를 생산 /소비

웹 3.0 
- 개인맞춤형 서비스

웹표준은 브라우져와 OS에 상관없이 동일한 UX를 제공하기 위함
2.0부터 복잡성이 증가

---------------------------------------------------------------------

개발 프로세스

1. 요구사항 분석
- 고객의 요구사항을 분석해 기능을 결정
- 개발 방향성 수립 및 원할한 협업

2. 설계
- 개발 환경 구성
- 데이터 구조 설계 및 관리
- 컴포넌트, 모듈간 연동

3. 개발
- API 나오기전에 모킹 데이터를 사용해서 와꾸먼저

4. 배포
- 개발 완료후 테스트 환경을 구성해 QA 진행
- QA가 없으면 개발자가 직접 검증까지 ㅠㅠ
- CI/CD와 호스팅 서비스를 조합해 배포 전략 수립
